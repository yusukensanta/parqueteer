# parqueteer Performance Benchmarks

Tracks wall-clock timings for core operations across file sizes and column counts.

## Methodology

- **Tool**: `scripts/benchmark.sh` (shell timing via `/usr/bin/time` or `date`)
- **Metric**: wall-clock elapsed time, cold JVM start per invocation
- **Iterations**: 5 runs, reporting min / avg / max
- **Data**: generated by `scripts/generate-test-data.sh`

### Why cold-JVM timing?

parqueteer is a CLI tool — each invocation starts a fresh JVM. JVM JIT warmup is
therefore part of the real workload. Unlike JMH (which excludes warmup), we measure
wall-clock from process start to exit. JIT effects average out over 5 runs.
